# Creating Presentation Page

学んだこと・試したことを**外部の読み手（発表・勉強会の参加者）**に伝えるHTMLレポートページを生成する。

ゴールは「読み手が読み終わったあとに、発表者の工夫・学び・ポイントが自分の言葉で語れる状態になること」。

---

## 依存

- `references/html-structure.md` — HTMLデザインガイド（本気AIスタイル・配色・コンポーネント）
- `references/exemplar.md` — 成功パターンのHTML例とページ構成
- キャラクター画像: `output/skill-journey/images/` 内の PNG ファイル

---

## ワークフロー

### Phase 0: ヒアリング

ユーザーに以下を確認する（まだ教えてもらっていない場合のみ）。

```
1. 発表・共有したいトピックは何か
2. 読み手に伝えたい「工夫・学び・ポイント」を3〜5つ挙げてもらう
3. 読み手像（どんな人が見るか、どのくらい詳しいか）
4. surge.shのドメイン名（例: my-topic-report.surge.sh）
```

ユーザーが既に内容を伝えている場合はこのPhaseをスキップする。

---

### Phase 1: ページ構成の設計

以下の標準4セクション構成を基本とし、トピックに合わせて調整する。

| セクション | 内容 | 必須？ |
|-----------|------|-------|
| 背景 | なぜこのトピックに取り組んだか | ✅ 必須 |
| 本論（2〜3セクション） | 伝えたい工夫・学び・ポイントを展開 | ✅ 必須 |
| まとめ | 持ち帰ってほしい3点 | ✅ 必須 |

**ページ冒頭の必須要素**:
- 「ざっくり一言」カード（30字以内で本質を一言）
- 背景セクションに「使う人 / 使う場面 / ゴール」の3カード

---

### Phase 2: キャラクター対話の設計

`references/html-structure.md` のキャラクター設定を参照し、以下の配置で対話を設計する。

| 配置場所 | 対話の目的 |
|---------|-----------|
| 背景セクション冒頭 | 読み手の「なぜ？」を代弁し、ページ全体の目的を提示 |
| 本論の節目（1〜2箇所） | 具体的な疑問・驚きを入れて理解を深める |
| まとめセクション | 学びを俯瞰し、読み手への一言で締める |

**ルール**:
- マジくん：読み手の疑問・驚きを代弁する（専門知識不要な言葉で）
- マスター：本質・Why・工夫の理由を説明する
- 対話は1セクションに1往復を基本とする

---

### Phase 3: HTML生成

`references/html-structure.md` のテンプレートを使ってHTMLを生成する。

**生成ルール**:
1. Tailwind CSS CDN、Lucide icon CDN を使用
2. 本気AIブランドカラー（グラデーションヘッダー）を適用
3. キャラクター画像は `images/` ディレクトリから参照（`references/html-structure.md` の画像パス一覧参照）
4. 絵文字禁止、Lucide iconのみ使用
5. セクションは `<section-card>` スタイルで白カード化
6. スティッキーナビ（右サイドバー）を配置

**出力先**: `output/{slug}/index.html`

---

### Phase 4: 批判的レビュー（サブエージェント）

**「発表会の参加者」視点**で、忖度なくレビューする。

```
Task({
  subagent_type: "generalPurpose",
  description: "発表ページの批判的レビュー",
  readonly: true,
  prompt: `
    あなたは「発表会に参加した社会人（30代以上、業種不問）」です。
    専門知識はなく、発表者が何を工夫したのかを理解したいと思っています。
    
    以下のHTMLファイルを読んで、忖度なく批判してください：
    
    【HTMLファイルパス】{path}
    
    ## レビュー観点
    
    1. **伝わらない表現**: 専門用語・略語で意味がわからない箇所
    2. **構成の不明瞭さ**: 「結局何が言いたいか」が追えない箇所
    3. **ざっくり一言の精度**: 30字以内で本質が伝わっているか
    4. **キャラクターの自然さ**: マジくんの疑問が読み手を代弁できているか
    5. **まとめの納得感**: 持ち帰れる3点が明確かつ行動につながるか
    
    ## 出力形式
    
    ### 致命的な問題（これがあると伝わらない）
    - ...
    
    ### 改善すべき点（直せばもっと良くなる）
    - ...
    
    ### 良かった点（このまま残すべき）
    - ...
    
    ### 総評
    5段階評価: ⭐⭐⭐☆☆（3/5）のように
    一言: 「○○だから△△な人には伝わりにくい」のように
  `
})
```

---

### Phase 5: ブラッシュアップ

Phase 4 のレビュー結果を踏まえて HTML を修正する。

**優先度**:
1. 致命的な問題 → 必ず修正
2. 改善すべき点 → 可能な限り対応
3. 良かった点 → 維持

---

### Phase 6: デプロイ

```bash
# 1. キャラクター画像をコピー
SLUG="{slug}"
OUTPUT_DIR="output/$SLUG"
mkdir -p "$OUTPUT_DIR/images"
cp output/skill-journey/images/*.png "$OUTPUT_DIR/images/"

# 2. surge.sh にデプロイ
cd "$OUTPUT_DIR" && npx --yes surge . --domain {ドメイン名}.surge.sh
```

## 品質チェックリスト

### 必須

- [ ] 「ざっくり一言」が30字以内で本質を表現している
- [ ] 背景セクションに「使う人 / 使う場面 / ゴール」の3カードがある
- [ ] マジくん＆マスターの対話が導入・本論・まとめに配置されている
- [ ] まとめに「持ち帰ってほしい3点」がある
- [ ] 専門用語には補足説明がある

### デザイン

- [ ] Lucide icon を使用（絵文字禁止）
- [ ] 本気AIブランドカラーのグラデーションヘッダーが適用されている
- [ ] キャラクター画像が正しく表示されている
- [ ] スティッキーナビがある（画面右サイドバー）
- [ ] スマホでも読みやすいレスポンシブ対応
