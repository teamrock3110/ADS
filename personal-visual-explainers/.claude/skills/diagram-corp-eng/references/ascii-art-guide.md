# ASCIIアートガイド

一枚絵サマリーは必ずASCIIアートで作る。
「読み始める前に全体像が頭に入る」ことが、理解の速度を決める。

---

## 使用する文字セット

```
ボックス: ┌ ─ ┐ │ └ ┘ ├ ┤ ┬ ┴ ┼
矢印:     → ← ↑ ↓ ↔ ▶ ◀
ラベル:   [ ] ( ) 半角スペースで整列
区切り:   ═══ ─── ···
状態:     ✓ ✗ ★ ※（スパリングに使う）
```

---

## パターン1: ネットワーク構成図

コンポーネントの配置関係・通信の流れを示す。

```
例: RADIUS認証のフェイルオーバー構成

  [デバイス]
      │
      │ 802.1X / EAP
      ▼
  [無線コントローラ / WLC]
      │                    ┌──────────────────┐
      │ RADIUS (UDP:1812)  │  認証サーバ群     │
      ├───────────────────▶│  [Primary]  ★稼働 │
      │                    │  [Secondary] 待機 │
      │  タイムアウト時:    └──────────────────┘
      └──────────────────────▶ [Secondary] に切替
```

```html
<!-- HTML表示例 -->
<div class="bg-slate-900 rounded-xl p-5 md:p-8 overflow-x-auto">
  <pre class="font-mono text-sm leading-relaxed text-emerald-400 whitespace-pre">
  [デバイス]
      │
      │ 802.1X / EAP
      ▼
  [無線コントローラ / WLC]
      │                    ┌──────────────────┐
      │ RADIUS (UDP:1812)  │  認証サーバ群     │
      ├───────────────────▶│  [Primary]  ★稼働 │
      │                    │  [Secondary] 待機 │
      │  タイムアウト時:    └──────────────────┘
      └──────────────────────▶ [Secondary] に切替
  </pre>
</div>
```

## パターン2: 時系列フロー（シーケンス図）

「誰が・何を・どの順で」行うかを左から右（または上から下）で示す。

```
例: SAML認証フロー

ユーザー          SP（サービス）          IdP（Okta等）
   │                   │                      │
   │  ① アクセス       │                      │
   │──────────────────▶│                      │
   │                   │  ② SAMLリクエスト    │
   │                   │─────────────────────▶│
   │                   │                      │ ③ 認証画面表示
   │◀─────────────────────────────────────────│
   │  ④ ID/PW入力      │                      │
   │─────────────────────────────────────────▶│
   │                   │  ⑤ SAMLアサーション  │
   │                   │◀─────────────────────│
   │  ⑥ アクセス許可   │                      │
   │◀──────────────────│                      │
```

## パターン3: 正常時 vs 障害時の比較フロー

2つの状態を縦に並べて差分を明確にする。

```
例: RADIUSフェイルオーバーの動作比較

【正常時】
WLC ──▶ Primary(1812) ──▶ Accept/Reject ──▶ 接続OK

【Primaryダウン時】
WLC ──▶ Primary(1812) ──×  （タイムアウト: timeout秒 × retries回）
    └──▶ Secondary(1812) ──▶ Accept/Reject ──▶ 接続OK
         ↑
         dead-time経過後は再びPrimaryを試みる
```

---

## パターン4: 設定値とフローの対応

設定パラメータがどこに影響するかを矢印で示す。

```
例: WLCのRADIUS設定値とフェイルオーバーの関係

  [timeout = 5秒]  ──── Primaryの応答待ち時間を制御
  [retries = 3回]  ──── タイムアウト後の再送回数を制御
  [dead-time = 10分] ── 障害判定後、Primaryをスキップする時間を制御

  フェイルオーバーまでの最大遅延:
  timeout × (retries + 1) = 5 × 4 = 20秒

  ┌─────────────────────────────────────────┐
  │  Primary                                │
  │  ① 送信 → タイムアウト(5s)             │
  │  ② 再送 → タイムアウト(5s)             │
  │  ③ 再送 → タイムアウト(5s)             │
  │  ④ 再送 → タイムアウト(5s) → 障害判定  │
  └─────────────────────────────────────────┘
  ↓ Secondary へ切替
```

## パターン5: コンポーネントの包含関係（階層）

「Aの中にBがあり、BはCを使う」構造を示す。

```
例: Okta Workforceの構成

┌─── Okta Org ─────────────────────────────────┐
│                                               │
│  ┌── Directory ──────┐  ┌── Applications ──┐  │
│  │  Users            │  │  Slack          │  │
│  │  Groups           │  │  Google WS      │  │
│  │  OU               │  │  Box            │  │
│  └──────────────────┘  └─────────────────┘  │
│           │                       │           │
│           └────────── SCIM ───────┘           │
│           └────────── SAML/OIDC ──┘           │
└───────────────────────────────────────────────┘
         ↕ AD Sync (LDAP / AD Agent)
┌─── オンプレ ActiveDirectory ──────────────────┐
│  DC01.corp.internal                           │
└───────────────────────────────────────────────┘
```

## ASCIIアートを書くときのルール

1. **幅を揃える**: 1行あたり60〜80文字以内を目安にする（横スクロールが起きない幅）
2. **ラベルは短く**: `[コンポーネント名]` は15文字以内
3. **矢印のラベル**: 矢印の上か下に、プロトコル名・設定値を書く
4. **状態の区別**: 正常は `★` または `[稼働]`、障害は `×` または `[ダウン]`
5. **段階は縦に**: フローが複数ステップある場合は縦方向に積む
6. **グループは囲む**: 同じ役割のコンポーネントは `┌──┐` で囲む

## HTMLでの表示

ASCIIアートは `<pre>` タグで表示する。フォントはモノスペース必須。

```html
<div class="bg-slate-900 rounded-xl p-5 md:p-8 overflow-x-auto">
  <pre class="font-mono text-sm leading-relaxed text-emerald-400 whitespace-pre">
  [ここにASCIIアートを書く]
  </pre>
</div>
```

Tailwind クラスで直接スタイリングする（`html-structure.md` のASCIIアートセクションも参照）。

## よくある間違い

| 間違い | 正しい書き方 |
|--------|------------|
| 日本語で幅が崩れる | 全角文字とボックス文字が混在すると崩れる。半角スペースで調整する |
| 矢印の向きが不明確 | `→` の代わりに `──▶` を使うと向きが明確になる |
| コンポーネントが多すぎる | 一枚絵は主要要素5個以内に絞る。詳細は後続セクションへ |
| フローが複雑すぎる | 正常時のみ → 異常時のみ → 比較、の3段階に分ける |
