# Gemini Gems セットアップガイド

## 前提条件

- **Gemini Advanced**（Google AI Pro / Ultra プラン）が必要

## セットアップ手順

### Step 1: Gem を作成

1. [gemini.google.com](https://gemini.google.com) にアクセス
2. 左サイドバーの「Gem マネージャー」→「新しい Gem を作成」
3. Gem 名: **「技術図解」**（任意の名前でOK）

### Step 2: 指示文を設定

1. 「指示」欄に `01_Gem指示文.md` の `---` 以降のテキスト全体を貼り付け

### Step 3: ファイルをアップロード

以下の4ファイルをアップロード（10ファイル上限の範囲内）:

| ファイル | 場所 |
|---------|------|
| `wpa2-vs-wpa3.html` | `output/wpa2-vs-wpa3.html` |
| `macbook-neo-vs-air.html` | `output/macbook-neo-vs-air.html` |
| `ascii-art-guide.md` | `output/claude-project-setup/ascii-art-guide.md` |
| `source-citation.md` | `output/claude-project-setup/source-citation.md` |

### Step 4: 保存 & テスト

「保存」して、以下のように入力してテスト:

```
SAML認証の仕組みを図解して
```

### HTMLの取得方法（Claude Projects との違い）

Gemini にはアーティファクト（HTMLプレビュー）がないため:

1. Gem がHTMLをコードブロックで出力する
2. コードブロックの「コピー」ボタンでコピー
3. ローカルに `index.html` として保存
4. ブラウザで開いてプレビュー確認
5. デプロイ:

```bash
npx --yes surge . --domain diagram-{トピック名}.surge.sh
```

### HTMLが途中で切れた場合

Gemini は長い出力を途中で切ることがあります。その場合:

```
続きを出力して。前回の出力の最後の行から再開してください。
```

## Claude Projects 版との差分

| 項目 | Claude Projects | Gemini Gems |
|------|----------------|-------------|
| HTML出力 | アーティファクト（プレビュー付き） | コードブロック（コピー→保存が必要） |
| 出力の安定性 | 途中で切れにくい | 長いHTMLは切れることがある |
| ファイル参照 | RAG経由 | 添付ファイル参照 |
| その他 | ほぼ同等 | ほぼ同等 |
