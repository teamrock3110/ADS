# Claude Projects セットアップガイド

diagram-corp-eng スキルを Claude Projects で再現するための手順書。

## セットアップ手順（5分で完了）

### Step 1: プロジェクト作成

1. [claude.ai](https://claude.ai) にアクセス
2. 左サイドバーの「Projects」→「+ Create Project」
3. プロジェクト名: **「技術図解 — コーポレートエンジニア」**（任意の名前でOK）

### Step 2: プロジェクト指示文を設定

1. プロジェクト設定画面で「Project instructions」を開く
2. `01_プロジェクト指示文.md` の **`---` 以降のテキスト全体** をコピー＆ペースト
3. 保存

### Step 3: ナレッジファイルをアップロード

1. プロジェクト設定画面で「Knowledge」→「Upload files」
2. 以下の3ファイルをアップロード:
   - `output/wpa2-vs-wpa3.html`
   - `.claude/skills/diagram-corp-eng/references/ascii-art-guide.md`
   - `.claude/skills/diagram-corp-eng/references/source-citation.md`

### Step 4: Web検索が有効か確認

プロジェクト設定で Web 検索（Search）がONになっていることを確認する。

### Step 5: 動作確認

プロジェクト内で新しいチャットを開き、以下を入力:

```
SAML認証の仕組みを図解して
```

確認ポイント:
- [ ] `text/html` アーティファクトでHTMLが生成されるか
- [ ] CSSがwpa2-vs-wpa3.htmlと同じスタイル（Apple風の黒白グレー交互）か
- [ ] 冒頭に「ざっくり一言」があるか
- [ ] ASCIIアートの全体像があるか
- [ ] インライン出典リンクがあるか（末尾リストではない）
- [ ] 品質チェック結果の自己申告テキストがあるか
- [ ] 途中でHTMLが切れていないか（フッターまであるか）

もしHTMLが途中で切れた場合は「続きを出力して」と送信する。

## 使い方

### 基本コマンド

```
○○を図解して
```

これだけで、Web検索 → 情報整理 → HTML生成 → セルフレビューが1ターンで実行される。

### FBして改善する場合

生成されたHTMLに対して、Cursorでやったのと同じようにFBを出せる:

```
FBです。
・ざっくり一言が抽象的すぎる。もっと具体的に
・ASCIIアートの矢印ラベルが英語のまま。日本語にして
・一次情報のURLが多すぎる。もっと絞って
```

### デプロイ（手動）

Claude Projects にはデプロイ機能がないため、手動で行う:

1. アーティファクトの「Copy」でHTMLをコピー
2. ローカルに `index.html` として保存
3. ターミナルで以下を実行:

```bash
# ファイルを保存したディレクトリで
npx --yes surge . --domain diagram-{トピック名}.surge.sh
```

## Cursor版との差分（何が変わるか）

| 項目 | Cursor版 | Claude Projects版 |
|------|---------|-------------------|
| **リサーチ** | サブエージェントが自動調査 | Claude のWeb検索が自動調査 |
| **HTML生成** | ファイルとして保存 | アーティファクトとして表示 |
| **レビュー** | 別サブエージェントが批判的レビュー | セルフレビュー（NG項目は修正してから出力） |
| **デプロイ** | 自動（surge.sh） | 手動（コピー→保存→surge.sh） |
| **FBループ** | チャット内でスキル自体を改善可能 | チャット内でHTMLを改善可能（指示文の改善は別途手動） |
| **デザイン再現** | ファイル直接読み込み（100%再現） | RAG経由（概ね再現、微差あり得る） |

### 品質差の見込み

- **コンテンツ品質**: ほぼ同等。Web検索の精度はCursorのWebFetchと遜色ない
- **デザイン品質**: 完成HTML実例をナレッジに入れることで概ね再現可能。ただしRAGが部分取得するため、CSSの一部が抜ける可能性あり。その場合は「CSSがwpa2-vs-wpa3.htmlと違う。合わせて」とFBすれば修正される
- **セルフレビュー品質**: Cursor版の別エージェントによるレビューより甘くなる可能性がある。重要な図解は自分の目でもチェックすることを推奨

## トラブルシューティング

### CSSが崩れる / デザインが違う

→ 「ナレッジのwpa2-vs-wpa3.htmlのCSSをそのまま使って。独自のCSSを追加しないで」と指示

### ASCIIアートが生成されない

→ 「全体像セクションにASCIIアートを必ず入れて。ナレッジのascii-art-guide.mdのパターンを参考に」と指示

### 一次情報URLが多すぎる / 少なすぎる

→ 「source-citation.mdの閾値ルールに従って。1カードにつき0〜2個が目安」と指示

### アーティファクトではなくコードブロックで出力される

→ 「HTMLをアーティファクトとして出力して」と指示
